Overview

Uses the arc4random algorithm to produce pseudo random numbers.

No manual seeding is required (or allowed) because arc4random self-seeds.

arc4random_uniform does not seem to work with Cocoa, so we replace with our own logic avoiding using % (modulo) and regenerating random integers when we need to (about once every million or so calls).

randomInteger

+ (NSInteger)randomInteger

Return Value

a positive or negative integer

Discussion

Generates a random integer between two large numbers. I thought range was ( –(232) – 1, (232) -1 ), but I find the range to be much larger when I unit test.

This is from the docs: The arc4random() function provides a high quality

                     32-bit pseudo-random number very quickly.

Suffice it to say that it produces a randomw integer between a very large negative integer and a very large positive integer.
